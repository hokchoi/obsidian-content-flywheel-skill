# 用 AI 终结收藏夹吃灰：Obsidian Content Flywheel Skill

这是一个面向内容创作者的 AI skill，用来把收藏夹、转录稿、网页长文、推特/X thread、小红书图文，转化成一个可持续高频输出的 Obsidian 内容生产系统。

## 一句话介绍

它不是让 AI 直接帮你写更多废稿，而是让 AI 先把你的输入变成可复用的知识原子、选题种子和多平台内容草稿。

## 它解决什么问题

很多内容创作者都有一个共同困境：

- 每天收藏很多链接，但真正创作时想不起来
- 看过很多爆款内容，但没有沉淀成自己的选题库
- Obsidian 里有很多笔记，但彼此之间没有形成可调用的网络
- AI 被用来写初稿，却没有参与更关键的输入处理和选题生成
- 同一个观点发不同平台时，只是复制粘贴，没有按平台重组

结果就是：收藏从未停止，创作从未开始。

## 这个 skill 怎么工作

它把内容生产拆成五个阶段。

### 1. 原始材料清洗

输入可以是：

- 长视频转录稿
- 播客文本
- 网页长文
- newsletter
- 推特/X thread
- 小红书图文
- 零散收藏链接

AI 会删除口水话、赞助商广告、重复铺垫和无效修辞，只保留观点、案例、数据、机制、方法和可以变成钩子的句子。

### 2. 四维知识矩阵

每条材料会被拆成四类知识原子：

1. 反直觉洞察：有没有打破常规认知的观点？
2. 情绪锚点：哪些案例、冲突或句子最容易引发共鸣？
3. 方法论框架：能不能变成读者立刻能用的步骤或模型？
4. 底层逻辑：现象背后的经济学、心理学、技术或运营机制是什么？

这一步的目的不是总结，而是把材料拆成未来可以反复调用的内容零件。

### 3. Obsidian 双链连接

新知识进入 Obsidian 后，skill 会搜索已有笔记，并按照这些关系提出双链建议：

- 同一个用户痛点
- 同一个底层机制
- 同一种案例类型
- 同一个工具或工作流
- 相反观点
- 可以继续追问的问题

每条链接都必须说明为什么连接，避免把知识库变成关键词噪音。

### 4. 选题种子生成

选题种子不是标题，而是一个待发布的论点。每个种子都会包含：

- 一句话论点
- 为什么现在值得讲
- 目标读者痛点
- 来源知识原子
- 关联历史笔记
- 平台适配度
- 还缺什么证据
- 这个选题可能变浅、变错、变无聊的风险

### 5. COPE 多平台重组

COPE = Create Once, Publish Everywhere。

同一个观点会被重组为不同平台的内容，而不是简单复制粘贴：

- X/Twitter、小红书：提取反直觉洞察和情绪锚点，生成短图文或 thread
- Newsletter、公众号、博客：组合方法论框架和底层逻辑，扩展成长逻辑文章
- 短视频、长视频、播客：转化为「黄金 3 秒开头 - 痛点展开 - 核心干预 - 证据案例 - 行动呼吁」

## 适合谁

这个 skill 适合：

- 高频更新内容账号的创作者
- 用 Obsidian 管理知识库的人
- 每天收藏大量内容但产出不稳定的人
- 希望用 AI 搭建内容工作流，而不是只让 AI 代写的人
- 想把一个观点稳定分发到小红书、X/Twitter、公众号、Newsletter、短视频或播客的人

## 仓库里有什么

```text
obsidian-content-flywheel-skill/
├── SKILL.md                  # skill 主文件
├── README.md                 # GitHub 仓库说明
├── templates/                # Obsidian Markdown 模板
├── references/               # 平台规则和 vault 结构说明
├── scripts/                  # 可选初始化脚本
├── examples/                 # 示例输入
├── evals/                    # 测试用例
└── docs/                     # 配套介绍文档
```

## 安装方式

把整个目录复制到本地 skills 目录，例如：

```bash
cp -R obsidian-content-flywheel-skill ~/.codex/skills/
```

也可以作为项目级 skill 使用，直接把 agent 指向 `SKILL.md`。

## 示例 prompt

```text
用 obsidian-content-flywheel 处理这段播客转录稿，输出知识矩阵和小红书草稿。
```

```text
把这个文件夹里的收藏链接做成选题种子库，并和我的 Obsidian 笔记建立双链建议。
```

```text
基于这篇长文生成 COPE 输出：推特 thread、小红书图文、公众号长文结构、短视频脚本。
```

## 核心观点

在人人都能使用大模型的时代，纯粹拼信息差会越来越弱。创作者真正的护城河不是「看过更多东西」，而是能不能把碎片信息变成自己的判断、连接和表达。

这个 skill 的目标不是替代创作者，而是移除从消费到创作之间最低价值、最重复、最消耗精力的部分。
