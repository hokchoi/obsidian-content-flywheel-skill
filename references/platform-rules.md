# Platform Rules

Use this file only when the user asks for platform drafts or COPE output.

## X / Twitter

- Lead with a compressed claim, not a topic label.
- Use short paragraphs.
- One thread should defend one idea.
- Avoid generic "here is why" phrasing unless the following line earns it.
- Best inputs: counterintuitive insight, mechanism, sharp example.

## Xiaohongshu

- Make the first line save-worthy or conflict-heavy.
- Use visual structure: short sections, clear bullets, image/slide suggestions.
- Good posts often combine emotional anchor with a usable method.
- Avoid sounding like a generic productivity article.
- Best inputs: emotional anchor, checklist, before/after contrast.

## Newsletter / WeChat

- Start from a thesis, then build context.
- Use cases from linked notes to avoid abstract commentary.
- Include objections or failure modes.
- Explain the mechanism behind the surface trend.
- Best inputs: method framework, underlying logic, historical or personal cases.

## Short Video

- The opening must create tension in the first three seconds.
- Use spoken language, not essay language.
- One video equals one conflict plus one intervention.
- Use examples that can be shown or acted out.
- Best inputs: emotional anchor, counterintuitive insight, one concrete action.

## Podcast

- Use a conversational arc: premise, story, mechanism, consequence, reflection.
- Allow more context, but avoid wandering.
- Use segment headers as recording anchors.
- Best inputs: underlying logic, case clusters, unresolved questions.
